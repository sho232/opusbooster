# レビュー

**レビューウィジェット** は、あなたのプラットフォームでのユーザーフィードバックとエンゲージメントを効率化するために設計された強力なツールです。 **カスタマイズ可能な5つ星評価システム** により、メンバーは簡単に体験を評価し、詳細なレビューを共有できます。

**主な機能：**

* **カスタマイズ可能なレビューフォーム** – レビューの進行方法の構成を定義します。
* **承認コントロール** – 任意の承認設定で新規レビューを管理します。
* **スレッド形式の返信** – レビュー内で構造化されたディスカッションを可能にします。
* **画像添付** – 画像でフィードバックを強化できるようにします。
* **スタイリングオプション** – 色、背景、表示レイアウトを調整します。
* **ライト＆ダークテーマ対応** – アクセシビリティとブランドの一貫性を担保します。

管理者は **コメントの順序、権限、可視性** を完全に制御でき、コミュニティのニーズに合ったシームレスな統合を実現します。インサイトの収集、エンゲージメントの向上、インタラクションの最適化など、 **レビューウィジェット** は動的で信頼性の高いフィードバック体験を生み出します。

{% hint style="success" %}
**ドラッグ＆ドロップ** – ページ、ファネル、ディスカッションスレッド内でレビューウィジェットを簡単に配置できます。
{% endhint %}

### レビューを作成

設定を開始するには、 **レビューウィジェット**、次の手順に従ってください：

{% stepper %}
{% step %}
左側のタスクバーに移動し、 **レビューウィジェットのアイコン** を見つけます。
{% endstep %}

{% step %}
ウィジェットを **ドラッグ＆ドロップ** でビルダーキャンバスに配置します。
{% endstep %}

{% step %}
設定の編集を開始して、レビューの表示方法と収集方法をカスタマイズします。

配置後、次のような詳細を微調整できます： **承認設定、コメントの可視性、スタイルの設定**
{% endstep %}
{% endstepper %}

![](<../../.gitbook/assets/image (34)>)

### 名前を入力

あなたの **レビューウィジェット** を異なるページ、ブログ、プロジェクトで簡単に再利用できるように、 **カスタム名またはラベル** を設定します。

* **一意の名前を定義** – ウィジェットを素早く特定して再利用するのに役立ちます。
* **コンテンツ間の一貫性** – ブランディングとレビュー構造を統一します。
* **将来の統合を簡素化** – 設定を再構成せずに新しいセクションへ簡単に挿入できます。

これにより、シームレスな **コンテンツ管理** が可能になり、 **一貫した体験** をレビューを配置する場所に関わらず実現します。

### 新規レビューに承認を要求

ユーザーフィードバックの品質管理を確保するために、 **承認** を新規レビューの公開前に要求できます。

仕組み：

* **承認を有効化** – この設定を有効にして、公開前に投稿を確認します。
* **管理コントロール** – モデレーターダッシュボードで、除外や削除を行えます。
* **レビューの健全性を維持** – ユーザーに表示される前に不適切または無関係なコンテンツを除外します。

この機能は、あなたのプラットフォームを **整理され、プロフェッショナルで、コミュニティ基準に沿った状態** に保ちます。

### 誰が返信できるか

もし **新しい返信に承認が必要** が有効な場合、レビューへのメンバー投稿のコメントには **管理者の承認** が公開前に必要になります。

仕組み：

* **コメントはキューに入れられ**、モデレーションダッシュボードで確認されます。
* **管理者が確認して承認** し、可視化される前に各コメントを精査します。
* **却下されたコメント** はユーザーに表示されません。
* **コンテンツの品質を保証** し、スパムや不適切な関与を防ぎます。

この機能により、より高い **ディスカッションのコントロール** が提供され、レビューのやり取りをプロフェッショナルで関連性の高いものに保ちます。

### 画像を許可

有効化すると、 **画像アップロード** を **レビュアー** と **レビューに返信するメンバー** の双方に許可でき、フィードバックにさらなる深みが加わります。

仕組み：

* **レビュアー** – レビューを補強するために画像を含めることができます。
* **返信するメンバー** – 返信内に画像を添付できます。
* **モデレーションコントロール** – 管理者は **承認または削除** を必要に応じて行えます。

この機能は **エンゲージメントと明瞭性** を高め、フィードバックをより視覚的で有益なものにします。

### 順序

ページ内でレビューを表示する順序を制御でき、あなたの構成に最適な流れを確保します。

* **昇順** – **最も古いレビューを先頭に** 表示し、最初から最新までの時系列を維持します。
* **降順** – **最新のレビューを先頭に** 表示し、直近のディスカッションを最上部で即座に確認できるようにします。

この設定を調整すると、 **エンゲージメントの優先順位付け** が可能になり、新しいやり取りを強調するか、時系列の読みやすさを維持するかを選べます。

### 最初に表示するレビュー数

このシンプルな設定により、 **レビューの表示数** を、メンバーがさらに読み込む前の初期状態で制御できます。

* 表示数は **3、5、10、または20** 件から選べます。
* これにより **クリーンで整理された** レイアウトを実現します。

この機能は、体験をスリムに保ちながら、柔軟なエンゲージメントを可能にします。

### テキストと背景のライトテーマ

この設定は、ダークモードまたはより暗い色の背景向けの調整です。

仕組み：

* **視認性の向上** – くっきりと読みやすいテキストでレビューを際立たせます。
* **デザイン調和の強化** – レビューが **レイアウトになじむ** ようにしつつ、アクセシビリティを維持します。
* **カスタムスタイリングオプション** – テキストの明るさや背景の透明度を調整して **最適なコントラスト**。

この機能により、 **レビューは視覚的に明瞭** でありながら、全体的なデザイン美に調和します。
