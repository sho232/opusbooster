# A/Bスプリットテスト

A/Bスプリットテストは、ウェブページやファネルの2つのバージョンを比較して、どちらがより良い成果を出すかを判断する方法です。ファネルにおけるA/Bスプリットテストでは、ファネル全体、または個々のファネルステップの異なるバリエーションを作成します。目標は、ニュースレターの登録や商品の購入など、コンバージョンの観点でどちらのバリエーションがより効果的かを見極めることです。

A/Bスプリットテストが非常に強力なのは、**データに基づく意思決定**ができる点です。どちらのバージョンが優れているかを推測する代わりに、テストの結果に基づいて判断できます。ファネルのどの要素が最も効果的かを特定することで、コンバージョンと収益を最大化するようにファネルを最適化できます。

ファネルのA/Bスプリットテストにあたっては、いくつかのベストプラクティスを押さえておきましょう。

1. **一度に1つの変数だけをテストする:** ファネルのどの要素がパフォーマンスの変化を引き起こしているかを正確に判断するには、一度にテストする変数を1つに絞ることが重要です。テスト対象は、コピー、デザイン、オファーなど、ファネルのあらゆる要素が考えられます。
2. **統計的有意性を確認する:** A/Bテストを実行する際は、結果が統計的に有意であることを確認しましょう。つまり、2つのバージョンの差が「意味のある差」と言えるほど十分に大きいことが必要です。
3. **継続的にテストする:** ファネルが本当の意味で「完成」することはありません。最善を尽くして最適化した後でも、改善の余地は常にあります。新しいアイデアを継続的にテストすることで、ファネルを常に最高の状態に保てます。
4. **結果を記録する:** A/Bテストの結果は継続的に記録しておきましょう。パターンや傾向を特定でき、今後ファネルをどう最適化するかについて、情報に基づいた判断ができるようになります。

A/Bスプリットテストは、ファネルを最適化してコンバージョンを高めるための強力なツールです。ベストプラクティスに従い、新しいアイデアを継続的にテストすることで、ファネルのパフォーマンスを常に最大限に引き出せます。

## スプリットテストの作成

A/Bスプリットテストでは、**ファネルステップのバリエーションを比較**してパフォーマンスを最適化できます。

* **ファネルステップに移動** – テストしたい**特定のステップ**を選択します。
* **グレーのバーにカーソルを合わせる** – **A/Bテスト**のオプションが表示され、設定を開始できます。
* **調整して比較** – ステップの異なるバージョンを作成し、どちらが最も良い結果を出すかを分析します。

**データに基づく改善**で、より良いコンバージョンにつなげましょう。

ステップを選択してA/Bテストをクリックすると、ポップアップが表示されます。ここでは2つのオプションから選べます。

* **ステップを複製**
* **新しいページ**

**ステップを複製**を選ぶと、ファネルステップ全体の完全なコピーが自動的に作成されます。コピーは必要に応じて編集できます（詳しくは後述します）。

**新しいページ**を選ぶと、あらかじめ用意されたページデザインを使ってページをゼロから作成できます。

この例では、**ステップを複製**オプションを選択して進めます。

## 最適なテストタイプの選択

冒頭のベストプラクティスでお伝えしたとおりです。

> **一度に1つの変数だけをテストする:** ファネルのどの要素がパフォーマンスの変化を引き起こしているかを正確に判断するには、一度にテストする変数を1つに絞ることが重要です。テスト対象は、コピー、デザイン、オファーなど、ファネルのあらゆる要素が考えられます。

スプリットテスト用のファネルステップに変更を加える前に、A/Bテストを選択すると表示される新しいポップアップを見てみましょう。この画面は次の2つの要素で構成されています。

1. **スプリットテストバー**
2. **ウィナー選択**

## スプリットテストバー

スプリットテストバーには、ステップAとステップB（つまりページ1とページ2）の両方が表示されます。バー内にはさまざまな数値が表示されますが、これらは各ページを訪れた人数とコンバージョン率、つまりそのファネルステップを訪れて次のステップに進んだ人の割合を示す分析データです。

## ウィナー選択

**ウィナー選択**は、ファネルの強力な機能のひとつです。次の2つのオプションから選択できます。

* **自動:** スプリットテストを自動化し、ファネルの最適化を任せられます。
* **手動:** 勝者となるステップを自分で選択できます。

## 自動ウィナー

**自動ウィナー**オプションを選択すると、ウィナー閾値（しきい値）となる数値を設定できます。これは「ファネルステップが一定の訪問者数に達したら、コンバージョンが最も多かった側を自動的に勝者として選択する」という意味です。

ヒント: 数値を決める際は、自分のファネルにどれくらいの人が訪れそうかを考えてみてください。オーディエンスが小さい場合は低めに、訪問者が多い場合は高めに設定します。始めたばかりであれば、100が良い出発点です。これで十分なデータと分析結果が得られ、**データに基づく意思決定**ができます。

## 手動ウィナー

**手動ウィナー**オプションでは、選択ボタンをクリックするだけで勝者を選べます。これは「ファネルステップが一定の訪問者数に達したら、訪問者数やコンバージョン率を見て自分で勝者側を選択する」という意味です。

注意: 自動・手動のどちらのオプションを使う場合でも、勝者を選択する（または自動で選ばれる）と、勝者にならなかった側のステップは削除されます。

## スプリットテストの編集

新しいスプリットテスト用のファネルステップを編集する際は、前述のとおり、小さな変更をテストすることでより精度の高いデータドリブンな洞察が得られます。では、具体的にどんな変更ができるでしょうか？

ファネルでスプリットテストできる要素の例・アイデアをいくつかご紹介します。重要なのは、一度に1つの変数に絞り、テスト結果が統計的に有意であることを確認することです。

| **スプリットテストのアイデア** | **説明** |
| ------------------ | ------- |
| ヘッドラインのコピー | 異なる見出しをテストして、どれがオーディエンスに最も響くかを確認しましょう。 |
| 本文のコピー | ヘッドラインと同様に、本文の異なるバリエーションをテストして、どれがよりコンバージョンにつながるかを確認しましょう。 |
| コールトゥアクション（CTA）のコピー | CTAはどのファネルでも重要な要素なので、正しく設定することが大切です。異なるバリエーションをテストして、どのCTAコピーが最も多くのコンバージョンを生むかを確認しましょう。 |
| CTAボタンのデザイン | CTAボタンのデザインもコンバージョンに大きく影響します。色、サイズ、形状を変えてテストし、どれが最も効果的かを確認しましょう。 |
| 画像 vs 動画 | ビジュアル要素がコンバージョンを大きく左右するファネルもあります。画像と動画のどちらがオーディエンスに効果的かをテストしてみましょう。 |
| フォームの項目 | ファネルにリード獲得フォームがある場合は、フォーム項目の異なるバリエーションをテストして、どれが最も多くのリードを獲得するかを確認しましょう。 |
| オファー | 商品やサービスを提供している場合は、オファーの異なるバリエーションをテストして、どれがオーディエンスに最も響くかを確認しましょう。 |
| 価格設定 | 商品やサービスを販売している場合は、異なる価格帯をテストして、どの価格が最も売上につながるかを確認しましょう。 |
| ページレイアウト | ページのレイアウトもコンバージョンに影響します。異なるレイアウトをテストして、どれが最も良いパフォーマンスを示すかを確認しましょう。 |

## テストの例

同じ見出しと同じコールトゥアクションを持ち、レイアウトデザインだけが異なる2つのページを比べてみてください。あなたなら、どちらをクリックしたくなるでしょうか？

ファネルのスプリットテスト機能の優れた点は、勝者を手動で選ぶか、自動化に任せて最適化させるかを自由に選べる柔軟性にあります。

通常のページからチェックアウトページまで、ファネル内のあらゆるステップをスプリットテストできます。

ただし、先にご紹介したベストプラクティスを忘れないでくださいね。
